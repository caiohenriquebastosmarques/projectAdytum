Boas Práticas de Código

                

                    
Princípio da Responsabilidade Única (SRP)

                    
Uma classe ou módulo deve ter apenas um motivo para mudar. Isso significa que deve ter apenas uma responsabilidade.

                    
Exemplo Ruim:

                    
// Classe que faz muitas coisas
class UserProcessor {
    processUserData(data) { /* processa */ }
    saveUserToDatabase(user) { /* salva no DB */ }
    sendWelcomeEmail(user) { /* envia email */ }
}

                    
Exemplo Bom:

                    
// Classes separadas
class UserDataProcessor { process(data) { /* processa */ } }
class UserRepository { save(user) { /* salva no DB */ } }
class EmailService { sendWelcome(user) { /* envia email */ } }


                

                

                    
Nomes Significativos

                    
Use nomes de variáveis, funções e classes que revelem a intenção e o propósito.

                    
Exemplo Ruim:

                    
let d; // o que é d?
function proc(arr) { /* ... */ }

                    
Exemplo Bom:

                    
let daysSinceModification;
function processUserList(userArray) { /* ... */ }
